# WanaWorks Product Roadmap and Execution Plan

## Current product direction

WanaWorks is a premium UE5 workspace platform and autonomous developer-assistance layer.

It should not feel like a normal Unreal plugin panel.

It should feel like:
- a software studio inside Unreal
- a non-destructive enhancement platform
- a character-intelligence wedge first
- a broader UE5 assistant platform over time

The immediate goal is an investor-demo-quality experience centered on Character Intelligence, Character Building, and Level Design, with a beautiful shell, a working subject preview, and a simple autonomous workflow.

## Product status

### Demo/prototype foundation
Approximate completion: 70%

### Full WanaWorks vision
Approximate completion: 38-40%

## Current major blockers

1. UI still feels too much like Unreal/Slate
2. workspace body switching is not mature enough
3. center live preview needs to feel alive and useful
4. Character Building needs to become a real workspace
5. Level Design needs to become a real WIT/world workspace
6. visible AI/character improvement needs to be more dramatic
7. UE5 assistant/generation layer has not started yet

---

# Phase 1 - Premium Studio Shell and Visual Identity

## Goal

Make WanaWorks look and feel like premium standalone software inside Unreal.

## Required qualities

- custom WanaWorks visual language
- not default UE tool styling
- strong left workspace rail
- center hero stage
- right live status modules
- bottom workflow strip
- polished typography
- premium cards
- minimal visible actions
- no button clutter
- no output log as main feedback

## Main visible actions

- Enhance
- Test
- Analyze
- Build

## Exit criteria

- screenshot no longer reads as a normal UE utility panel
- main UI feels branded and investor-demo worthy
- no visible workflow tool pile
- output/logs are secondary
- Build is quiet and non-intrusive

# Phase 4 - Autonomous Subject Workflow

## Goal

Remove manual setup friction.

## User flow

1. Pick AI Pawn or Character Pawn from inside WanaWorks
2. WanaWorks detects stack automatically
3. WanaWorks creates working copy automatically when needed
4. WanaWorks enhances, tests, analyzes, and builds

## WanaWorks must detect

- pawn type
- linked Anim BP
- skeletal mesh
- AI controller
- AI-readiness
- WanaWorks components
- animation integration status
- physical state status
- behavior status

## Exit criteria

- no need to select actor in UE viewport as primary path
- no need to manually create copy
- no need to manually wire supported integrations
- original assets preserved

---

# Phase 5 - Automatic Anim BP Integration

## Goal

Remove manual Anim BP wiring for supported workflows.

## Requirements

- auto-detect linked Anim BP from selected pawn
- auto-attach WanaWorks integration component to generated/working/finalized subjects
- auto-wire supported state when possible
- original Anim BP untouched
- support boundaries clearly reported

## Integration states

- Ready
- Applied
- Limited
- Not Supported

## Exit criteria

- supported cases do not require opening Anim BP manually
- UI reports auto-attach and auto-wire state
- original source assets remain untouched
- unsupported cases fail clearly and beautifully

# Phase 6 - Visible Character / AI Improvement

## Goal

Create a dramatic enough visual improvement that the demo feels real.

## Target slice

Impact -> Physical State -> Animation/Behavior Response

## Required systems

- physical state
- instability alpha
- directional impact
- recovery progress
- bracing
- movement/attack commitment flags
- animation integration
- behavior result

## Exit criteria

A normal viewer can see:
- medium impact causes visible disruption
- stronger impact causes stronger off-balance/recovery
- subject behaves differently after WanaWorks enhancement
- result is obvious without reading logs

# Definition of demo-ready

A demo-ready WanaWorks build must show:

1. Premium software shell
2. Subject picked from inside WanaWorks
3. Stack auto-detected
4. Subject appears in live preview
5. Enhance applies automatically
6. Test shows visible behavior/animation/physical state result
7. Build creates clean output asset
8. Original source is preserved
9. UI looks beautiful enough to show investors

If any of these are missing, the demo is not ready.
